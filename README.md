## 🎋 Merge 전략

### Issue 사용
1. 구현하고자 하는 기능 issue 생성
2. main`feat/#이슈번호` 브랜치 생성 
3. 기능 구현 후 커밋
4. 해당 브랜치에서 git push
5. 깃허브에서 PR 생성
7. 충돌 없으면 Main에 Merge

### GIT 컨벤션
- main
- feat/#10
- hotfix/#4

## CICD 전략
- GitAction를 이용해 main 브랜치에 Merge함과 동시에 배포
- 프론트와 빠른 협업을 위함
